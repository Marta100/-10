1)რა განსხვავებაა let, const შორის?
let- მნიშვნელობა შეიძლება შეიცვალოს, const-უცვლელია

 2)რა განსხვავებაა primitive და nonprimitive type-ებს შორის?
* პრიმიტიული -გადაეცემა მნიშვნელობით, ( თვითონ მნიშვნელობა). 
 პრიმიტიულ ტიპებია: string, number, boolean,  null, undefined, symbol, bigint.
* არაპრიმიტიული- გადაეცემა მითითებით, იზიარებს ერთ მეხსიერებას (მისამართი სადაც მნიშვნელობა ინახება).
 არაპრიმიტიული ტიპებია: object, arrey, function.

3)რა განსხვავებაა == და === ოპერატორებს შორის?
== შეამოწმებს მხოლოდ მნიშვნელობას, === შეამოწმებს მნიშვნელობას და ტიპს.

4)რა ტიპის არის null
null არის პრიმიტიული ტიპი, მაგრამ typeof მას შეცდომით "object" აბრუნებს. ის არ არის ობიექტი,
 ის ჯავასკრიპტის ძველი შეცდომაა, რომელიც დღემდე დარჩა. null ნიშნავს განზრახ ცარიელს, ანუ აქ არაფერია.

5)რა განსხვავებაა funtion და = () => {} შორის ყველაზე მნიშვნელოვანი განსხვავება: this-ის ქცევა.
function-ს აქვს საკუთარი this, arguments და შეიძლება იყოს new-ით constructor.
() => {} (arrow function)-ს საკუთარი this და arguments არ აქვს, და constructor ვერ იქნება.
Arrow function უფრო მოკლე სინტაქსია და ხშირად callback-ებისთვის გამოიყენება.

6)რას ნიშნავს callback function? 
ეს არის ფუნქცია რომელიც გადაეცემა სხვა ფუნქციას არგუმენტის სახით და გამოიძახება 
მას შემდეგ რაც მთავარი ფუნქცია თვის საქმეს დაასრულებს.

7)რა განსხვავებაა map, filter და reduce-ს შორის?
სამივე მუშაობს მასივზე, map(შეცვალე) გარდაქმნის თითოეულ ელემენტს, filter(ამოარჩიე) გაფილტრავს, ზოგიერთს ამოიღებს,
 reduce (შეაჯამე) აერთიანებს ყველაფერს ერთ მნიშვნელობად.

 8)რა არის Promise?
 არის ობიექტი რომელიც ასინქრონული ოპერაციის საბოლოო შედეგს ასახავს, მას აქვს 3 მდგომარეობა:
 pending მოლოდინი,(ოპერაცია მიმდინარეობს)
fulfilled წარმატება (ბრუნდება შედეგი და მუშავდება .then()ით) 
rejected შეცდომა ( ბრუნდება მიზეზი და მუშავდება .catch() ით).

9)რა განსხვავებაა Promise.all, Promise.race და Promise.any-ს შორის?
ეს სამი მეთოდი გამოიყენება მაშინ როცა ერთდროულად რამდენიმე promise-თან გვაქვს საქმე
 და მათი ერთობლივად მართვა გვინდა.
 Promise.all ყველა ერთად: ელოდება ყველას წარმატებას,თუ ერთი მაინც ჩავრდა, ყველაფერი ფუჭდება.
 Promise.race შეჯიბრია: აბრუნებს იმას , ვინც ყველაზე სწრაფია(სულ ერთია წარმატება იქნება თუ შეცდომა),
 Promise.any პირველივე წარმატება: აბრუნებს ყველაზე სწრაფ წარმატებულს. შეცდომებს აიგნორებს თუ ყველა არ ჩავარდა.

 10)რას ნიშნავს async/await და რატომ ვიყენებთ?
 ეს არის სინტაქსი რომელიც ასინქრონული კოდის წერისას უფრო მარტივს, სუფთას და ვიზუალურად სინქრონულს ხდის.
 async  იწერება ფუნქციის წინ. ეს ნიშნავს, რომ ფუნქცია ავტომატურად ყოველთვის აბრუნებს promise-ს.
 await  იწერება მხოლოდ async ფუნქციის შიგნით და აიძულებს კოდს, დაელოდოს ასინქრონული ოპერაციის დასრულებას,
  სანამ მომდევნო ხაზზე გადავა.
   ვიყენებთ იმიტომ რომ კოდი იყოს უფრო სუფთა და მოკლე.

 11)რა განსხვავებაა class-სა და object-ს შორის?
 class შაბლონია რომლის მიხედვითაც იქმნება ობიექტები.
 object ამ შაბლონის კონკრეტული ეგზემპლარია, რომელიც რეალურ მონაცემებს ინახავს. ობიექტი არის კონკრეტული 
 მონაცემებისა და ფუნქციების ერთობლიობა.

 12)რას აკეთებს constructor class-ში?
 კონსტრუქტორი კლასში აკეთებს 2 რამეს:
 1. ქმნის საწყის თვისებებს: ახალ ობიექტს ანიჭებს პარამეტრებს.
 2. ავტომატურად ირთვება: მუშაობს მხოლოდ ერთხელ new ოპერატორის გამოყენებისას.
 მოკლედ ის არის ობიექტის დამზადების და მისი საწყისი მონაცემებით შევსებს ფუნქცია.

13)რა განსხვავებაა მემორი ჰიპსა და მემორი სტექკს შორის
javascript-ში მეხსიერება იყოფა ორ ნაწილად:
memory heap ინახავს რთულ ტიპებს (objects, arrays, functions) არის დიდი, დინამიური და მონაცემები მასშიქაოსურადაა მიმოფანტული.
memory stack ინახავს მარტივ ტიპებს (promitives: number, string, boolean) და ობიექტების მისამართებს. არის ძალიან სწრაფი,
 პატარა და აქვს მკაცრი სტრუქტურა.

14) რა არის ქოლსტეკი
call stack  არის ჯავასკრიპტის მექანიზმი, რომელიც ადევნებს თვალს ფუნქციების სესრულების რიგითობას.
ის განსაზღვრავს , თუ რომელი ფუნქცია მუშაობს მოცემულ მომენტში და რა ფუნქციები უნდა შესრულდეს მის შემდეგ.

15) რა არის ივენთ ლუპი
event loop არის ჯავასკრიპტის შიდა მექნიზმი, რომელიც უზრუნველყოფს კოდის ასინქრონულ მუშაობას. მარტივად რომ ვთქვათ
ეს არის მორიგე, რომელიც ელოდება ქოლსტექის დაცარიელებას რათა რიგში მდგომი ფონური დავალებები სათითაოდ შეუშვას სამუშაოდ.
